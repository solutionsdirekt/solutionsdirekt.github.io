Azure DevOps Caching für Powershell Module

Azure DevOps besitzt eine mächtige Möglichkeit Dateien zwischen Builds zu cachen und somit Build-Zeiten zu verkürzen

Stefan Schoof
8 October 2020

# Problem

Für viele Builds müssen Abhängigkeiten installiert werden. Diese Installationen benötigen oft einiges an Zeit.

# Lösung

Azure DevOps hat seit einiger Zeit ein Feature namens Pipeline caching. Dieses Feature erlaubt es einen Ordner in einem Cache außerhalb des Agents abzulegen. Der Cache bekommt einen Cache Key, der anhand von Identifiern, Dateihashes und Variablen gebildet werden kann.

Ich hatte gerade den Fall, dass für einen Build eine Reihe von Powershell Modulen installiert werden musste. Dies dauert an guten Tagen 5 Minuten pro Build.

Mit einem recht einfachen Yaml Snippet lässt sich das Caching in die Pipeline einbauen:

    - job: JobName
      pool: Hosted VS2017
      steps:
      - task: Cache@2
        inputs:
          key: powershell-modules | install-modules.ps1
          path: C:\Users\VssAdministrator\Documents\WindowsPowerShell\Modules
          cacheHitVar: CACHE_RESTORED

      - task: PowerShell@2
        displayName: Install Modules
        inputs:
          filePath: install-modules.ps1
        condition: ne(variables.CACHE_RESTORED, 'true')

Solange die Datei install-modules.ps1 sich nicht verändert wird der Install Modules Task nicht ausgeführt. Der Cache reduziert die Zeit auf 10 Sekunden.

Der Cachekey wird hier anhand des Fixen strings powershell-modules und dem Hash der Datei install-modules.ps1.

Da es nur ein Cache ist, kann es sein das es noch einzelne Builds gibt, die install-modules.ps1 ausführen müssen, aber für die meisten Fälle ist es deutlich schneller. Dies kann z.B. an der Cache isolation oder weil der Cache nicht mehr verfügbar ist.

Der Cache Task fügt automatisch eine Post job ein. In diesem wird der Inhalt des Ordners unter dem Cachekey hochgeladen.

# Ausblick

Mit so einem Cache lassen sich viele Pipeline verbessern. Bei meinen Experimenten mit npm war leider kein durschlagender Erfolg zu sehen. Da die npm Packages aus so vielen Dateien bestehen, benötigt der Cache Restore leider auch sehr viel Zeit.
